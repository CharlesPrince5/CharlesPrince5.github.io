V-Suite CORE连接

V-Suite CORE连接

为了将数据从V-Suite CORE传输到V-Suite Web，您必须配置数据传输作业。您必须为每个V-Suite CORE工程创建一个数据传输作业。同一服务器上项目的V-Suite CORE连接信息始终相同，因此V-Suite允许您添加可用于该服务器的所有数据传输作业的别名。


请考虑以下示例，其中在服务器“MY_VES_SERVER”上为V-Suite CORE安装 Visionaize Enterprise Server (VES)， V-Suite CORE数据库实例名为“MY_SQL_INSTANCE”并安装在服务器“MY_SQL_SERVER”上，并且 SQL 配置为监听 1500 端口。


输入以下信息以配置V-Suite CORE连接别名：

V-Suite CORE连接：

  • 别名- MY_VES_SERVER（或任何唯一的描述性名称）
  • V-Suite CORE SQL Server 名称或IP - MY_SQL_SERVER
  • V-Suite CORE SQL 实例名称- MY_SQL_INSTANCE。对于默认实例，将其留空。
  • TCP 端口- 1500。将默认实例的端口号留空，输入自定义端口的端口号。
  • V-Suite CORE服务器名称- MY_VES_SERVER
  • V-Suite CORE内置“admin”密码  - 输入您用于以内置“管理员”用户身份登录 VES 的密码。
  • V-Suite CORE版本- 选择您正在使用的V-Suite CORE版本，例如， V-Suite 2017 选择 7.0。您可以在V-Suite CORE应用程序中查看版本信息。


按测试连接按钮以验证您是否输入了有效的连接信息。

按“保存”按钮保存新的V-Suite CORE连接别名。


排除连接错误

当您在测试 CORE 连接时收到连接错误时，请尝试以下步骤来解决该错误：

 

    1. 验证输入的连接参数是否正确 - 确保输入的名称中没有拼写错误，并且所选的 CORE 版本与目标服务器上安装的V-Suite CORE版本匹配。另外，请仔细检查内置的“admin”密码是否输入正确。
    2. 确保目标 SQL Server 实例可访问，即它不会因维护而停机。
    3. 在某些情况下， V-Suite Data Transfer 可能无法连接到V-Suite CORE SQL Server。请联系您的系统管理员以验证是否满足V-Suite数据传输要求。
