登入

要使用V-Suite Web 应用程序，需要登录系统。仅向有效的注册用户授予访问权限。未经身份验证的用户将被重定向到V-Suite登录页面。有关如何添加新用户的信息，请访问添加用户主题。


系统支持两种验证用户的方法：

  • 单点登录 (SSO) 身份验证
  • V-Suite （基于密码）身份验证


您只能使用系统管理员启用的身份验证方法。


单点登录 (SSO) 身份验证

以下步骤描述了 Azure AD 作为 SSO 提供程序时的 SSO 身份验证过程。请注意，Active Directory 联合身份验证服务 (ADFS) 的流程遵循非常相似的流程。

  • 要使用外部 SSO 提供商 (Azure AD) 登录，请单击“使用公司帐户登录”按钮。
  • 您将被重定向到 Azure AD 登录页面。如果你已经登录到 Azure AD（例如，从另一个应用程序），则可能看不到此页面。
  • 如果您看到 Azure AD 登录页面，请输入您的 Azure AD 用户名（通常是您的工作电子邮件）、Azure AD 密码，然后单击“登录”。如果您的组织已启用多重身份验证 (MFA)，请按照提示提供额外验证。
  • 成功通过 Azure AD 身份验证后，您将被重定向回V-Suite Web 应用程序。如果您的 Azure AD 帐户链接到V-Suite用户帐户，您将获得对V-Suite应用程序的访问权限。
  • 如果您的 Azure AD 帐户未链接到V-Suite用户帐户，您将看到一条错误消息，提示您联系V-Suite管理员或以其他用户身份登录。


V-Suite身份验证

要使用V-Suite身份验证登录，请选择“使用V-Suite帐户登录”按钮。然后系统将提示您输入V-Suite凭据：

  • 电子邮件 - 输入与您的V-Suite帐户关联的电子邮件地址。例如：“johndoe@mydomain.com”。
  • 密码 - 提供您的V-Suite帐户的密码。


登录 - 更改密码

当您首次登录V-Suite或尝试使用过期密码登录V-Suite时，系统会要求您输入新的V-Suite帐户密码。要更改密码，您需要输入以下信息：

  • 旧密码 - 您的V-Suite帐户的旧（过期）密码
  • 新密码 - V-Suite帐户的新密码。密码必须符合密码复杂度设置。
  • 确认新密码 - 再次输入所需的新密码进行确认。


输入密码信息后，单击“更改密码”按钮将更改您的密码并登录V-Suite网站。


保持登录状态

当您选择“保持登录状态”复选框时，即使您关闭浏览器，您的信息也会被记住长达 30 天。 30 天内下次打开同一浏览器时，您将自动登录。为了让“记住我”发挥作用，您不应单击应用程序中的“退出”按钮，而只需在完成后关闭浏览器即可。

通过使用“保持登录状态”选项，您可以更轻松地再次登录，但这也意味着有权访问计算机（使用同一帐户访问计算机）的任何人都将能够使用您的V-Suite登录帐户。请勿在公共计算机或您无法控制访问的任何其他地方使用“记住我”。


V-Suite管理员可以完全禁用“保持登录状态”功能或调整记住登录信息的天数（默认为 30 天）。

登录 - 错误

下表列出了您在登录时可能遇到的错误消息：

错误信息

原因

用户名或密码无效

  • 您输入了错误的电子邮件和/或密码
  • 您不是有效的V-Suite用户。请联系您的系统管理员。

由于密码错误次数过多，您的V-Suite帐户被阻止登录。请联系您的系统管理员。

  • 您输入错误密码的次数过多。请联系您的系统管理员。

您的帐户已禁用，请联系您的系统管理员。

  • 您的V-Suite帐户已禁用。请联系您的系统管理员。

您需要更新密码，因为这是您第一次登录，或者您的密码已过期。

  • 您需要更改您的V-Suite用户帐户密码。

单点登录错误 - 此帐户当前无权访问V-Suite。如果您认为应该授予您访问权限，请联系您的V-Suite管理员寻求帮助。

  • SSO 用户不是有效的V-Suite用户。请联系您的系统管理员。
