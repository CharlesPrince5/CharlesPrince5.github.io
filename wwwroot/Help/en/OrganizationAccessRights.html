Organization Access Rights

About access rights

V-Suite users must be given access rights to data. In order to assign access rights to a user, you must assign the user to an Organization Role. The role is assigned to items in your organization hierarchy, granting the members the right to see the organization items. The organization hierarchy is transferred from V-Suite CORE and currently supports the V-Suite CORE "Project" and "Workspace" level. 

The following is example hierarchy where the "Project Supervisor" role is assigned to the "Project" level and the Inspection, Maintenance, and Operations Engineer roles are assigned to the "Workspace" levels.

In V-Suite Web, the following restrictions apply:

  • Assigning a role to an organization item will assign the "View" access right. "Modify" and "Delete" access rights are not supported.
  • The organization hierarchy supports Knowledge Book folders only
  • Assigning the view access right to the "Project" level is inherited by the "Workspaces" under the project.
  • You cannot assign a role to both the project and workspace level.

Manage organization access rights

Organization Roles are managed via the Organization Access Rights application. The Organization Access Rights application can be accessed via the Project Administration icon on the V-Suite home page.



You will not see the organization access rights application if you are not entitled to use the application. Click here to learn more about entitlements.


The application presents a table with the V-Suite Knowledge Book organization hierarchy. The table will present the "Project" organization items. Click the project organization item to navigate to the workspace organization items under the project.

You can assign or edit a role to an organization item.

In the organization access right  table, select the organization item you wish to edit and click the Edit icon in the Action column. This displays the Assign Role form which lets you assign or remove roles from the organization item.
