要件

V-Suite 3D Web アプリケーションを使用するときに最適な GPU パフォーマンスを確保するには、以下の推奨事項を参照して、お使いの構成と比較してください。

ブラウザ

V-Suite は現在、次のテスト済みおよび承認済みのブラウザをサポートしています。

  • モジラ Firefox
  • グーグルクローム
  • マイクロソフトエッジ

Mac OS Safari は現在サポートされていません。

Microsoft が Internet Explorer (IE) 11 のサポートを終了したため、 V-Suite WEB クライアントは Microsoft Internet Explorer 11 をサポートしなくなりました。


WebGL

V-Suite 3D Web およびモバイル アプリケーションには WebGL が必要です。ブラウザで WebGL が有効になっていることを確認してください。 Chrome では、グラフィック カードが WebGL をサポートしていない限り、デフォルトで有効になっている必要があります。

Chrome で WebGL を有効にする

  • chrome://flags に移動します
  • 「WebGL を無効にする」が有効になっていないことを確認します (変更を有効にするには Chrome を再起動する必要があります)。

Firefox で WebGL を有効にする

  • about:config に移動します
  • WebGL で検索します。無効
  • 値が false であることを確認します (変更は Firefox を再起動しなくてもすぐに有効になります)。


Microsoft Edge で WebGL を有効にする

  • Edge://settings/system に移動します
  • 「利用可能な場合はハードウェア アクセラレーションを使用する」にチェックが入っていることを確認します。ブラウザを再起動します。
  • Edge://GPU に移動します
  • [グラフィックス機能のステータス] で WebGL を見つけ、[ハードウェア アクセラレータ] がリストされていることを確認します。

ノート

グラフィックス ドライバーが古すぎて WebGL をサポートできない場合は、アップグレードできる可能性があります。

詳細については、こちらをご覧ください。

  • Chrome: グラフィックが正しく表示されない
  • Firefox: グラフィック ドライバーをアップグレードする

ブラックリスト

特定のグラフィックス カードとドライバーは、重大なバグを含んでいたり、クラッシュを引き起こすことが知られているため、一部のブラウザーによってブラックリストに登録されています。非常に深刻な場合には、コンピュータ全体がフリーズする可能性もあります。

多くのブラウザには、ブラックリストをオーバーライドするオプションがあります (つまり、リスクにもかかわらず、ブラックリストに登録されている機能の使用を試みます)。これはお勧めしません。リスクを理解して受け入れる場合にのみ、ブラックリストを上書きしてください。

詳細については、 WebGL ブラックリスト (https://www.khronos.org/webgl/wiki/BlacklistsAndWhitelists) を参照してください。
