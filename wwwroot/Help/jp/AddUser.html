ユーザーを追加する

ユーザーがV-Suiteにサインインできるようにするには、まずユーザーをシステムに追加する必要があります。


ユーザー アプリケーションから新しいユーザーを追加できます。ユーザー アプリケーションには、 V-Suiteホームページのシステム管理アイコンからアクセスできます。



新しいユーザーのアイコンをクリックします。これにより、ユーザーのプロパティを指定できる「新規ユーザー」フォームが表示されます。

ユーザープロパティ

フォームに表示されるプロパティは、設定されたAuthentication Modeによって決まります。システムの構成された認証モードに関連するプロパティのみにアクセスできます。


新しいユーザー情報の入力の開始は、システムがシングル サインオン (SSO) プロバイダーで構成されているかどうかによって異なります。

  • システムがSSO プロバイダーを使用して構成されている場合 - まず、新しいユーザーの外部 IDを選択する必要があります。完了したら、残りのユーザー プロパティの入力に進むことができます。
  • システムがV-Suiteパスワード認証のみを使用している場合 -ユーザー プロパティを直接入力して開始できます。


外部アイデンティティ

システム構成でシングル サインオン (Azure AD または ADFS) がアクティブ化されている場合、次のプロパティが使用可能になります。

  • Azure AD 認証/ADFS 認証 -デフォルトでは、このオプションは有効になっています。ユーザーがシングル サインオン (SSO) を使用してサインインできないようにするには、チェックを外します。
  • Azure AD アカウント ユーザー名 / ADFS アカウント ユーザー名- このフィールドには、ユーザー アカウントに関連付けられた外部 (Azure AD または ADFS) ID のユーザー名が表示されます。外部 ID をリンクするには、 「ディレクトリの検索」ボタンをクリックします。この操作により、 [アカウントの選択]ダイアログが開き、関連付ける外部ユーザー ID を検索して選択できます。


ユーザープロパティ

外部ユーザー ID (SSO 対応システムの場合) を選択した後、またはV-Suite認証のみを使用する構成で、次のユーザー プロパティを編集できます。

  • 電子メール -ユーザーの電子メール アドレス。この電子メール アドレスは、V-Suite のパスワードベースの認証を介してシステムにサインインするためのユーザー名としても機能します。このフィールドは必須であり、一意である必要があります。
  • V-Suiteユーザー パスワード- V-Suiteアカウントのパスワード。パスワードはパスワードの複雑さの設定を満たしている必要があります。パスワードの有効期限はパスワード ポリシー設定に従います。 V-Suite のパスワード ポリシーの詳細については、ここをクリックしてください。
  • セキュリティ グループ- ユーザーが所属するセキュリティ グループを選択します。権限とアクセスの詳細については、 「資格」を参照してください。
  • 組織の役割- 組織の役割をユーザーに割り当てます。詳細については、 「組織の役割」セクションを参照してください。

[ユーザー システム設定]フォームの[セキュリティ グループ]および[組織の役割]フィールドに使用されるデフォルトを変更できます。

  • アカウント ステータス- [有効] または[無効]に設定できます。ユーザーが一時的または無期限にサインインできないようにする場合は、「無効」を選択します。

  • 名- ユーザーの名。このフィールドはオプションであり、空白のままにすることもできます。
  • 姓- ユーザーの姓。このフィールドはオプションであり、空白のままにすることもできます。


フォームの上部にある[保存]ボタンをクリックして、新しいユーザーを追加します。
