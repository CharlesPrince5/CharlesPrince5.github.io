ユーザーの編集

場合によっては、既存のユーザーの編集が必要になることがあります。たとえば、ユーザーの姓名を修正したり、アカウントを (一時的に) 無効にする必要がある場合があります。


ユーザー アプリケーションを介してユーザーを編集できます。ユーザー アプリケーションには、 V-Suiteホームページのシステム管理アイコンからアクセスできます。



ユーザー表で、編集するユーザーを選択し、「アクション」列の「編集」アイコンをクリックします。これにより、選択したユーザーのプロパティを変更できる「ユーザーの編集」フォームが表示されます。

フォームに表示されるプロパティは、設定されたAuthentication Modeによって決まります。システムの構成された認証モードに関連するプロパティのみにアクセスできます。


外部アイデンティティ

システム構成でシングル サインオン (Azure AD または ADFS) がアクティブ化されると、外部 ID プロパティが使用可能になります。

シングル サインオン (SSO) を禁止するように構成されたユーザー アカウントを、後で SSO を許可するように変更することはできません。したがって、ユーザーの SSO 設定は、新しいユーザーの作成時にのみ決定でき、後で変更することはできません。

  • Azure AD アカウント ユーザー名 / ADFS アカウント ユーザー名- このフィールドには、ユーザー アカウントに関連付けられた外部 (Azure AD または ADFS) ID のユーザー名が表示されます。別の外部 ID をリンクするには、 「ディレクトリの検索」ボタンをクリックします。この操作により、 [アカウントの選択]ダイアログが開き、関連付ける外部ユーザー ID を検索して選択できます。


ユーザープロパティ

次のユーザー プロパティを編集できます。

  • 電子メール -ユーザーの電子メール アドレス。この電子メール アドレスは、V-Suite のパスワードベースの認証を介してシステムにサインインするためのユーザー名としても機能します。このフィールドは必須であり、一意である必要があります。
  • V-Suiteユーザー パスワード- V-Suiteアカウントのパスワード。ユーザーに新しいパスワードを強制的に設定するには、 「パスワードの変更」ボタンをクリックします。パスワードはパスワードの複雑さの設定を満たしている必要があることに注意してください。パスワードの有効期限はパスワード ポリシー設定に従います。 V-Suite のパスワード ポリシーの詳細については、ここをクリックしてください。
  • セキュリティ グループ- ユーザーが所属するセキュリティ グループを選択します。権限とアクセスの詳細については、 「資格」を参照してください。
  • 組織の役割- 組織の役割をユーザーに割り当てます。詳細については、 「組織の役割」セクションを参照してください。

  • アカウント ステータス- [有効] または[無効]に設定できます。ユーザーが一時的または無期限にサインインできないようにする場合は、「無効」を選択します。

  • 名- ユーザーの名。このフィールドはオプションであり、空白のままにすることもできます。
  • 姓- ユーザーの姓。このフィールドはオプションであり、空白のままにすることもできます。


フォームの上部にある[保存]ボタンをクリックして、変更をユーザーにコミットします。
